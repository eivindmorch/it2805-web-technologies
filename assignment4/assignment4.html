The good website - Frysja Legekontor

EasyChecks

Title
The title changes per web view. The name of the office should be before the current webpage title.

Alt-text
Every picture has an appropriate alternative text.

Headings
The navigation used on the page can function as a heading, which lesser the need for a <h1> heading. They highlight the webpage you are visiting. The <h1> tag is used in the source code but is never used on the actual page. They only use the tag <h3> on the page.

Contrast
Contrast levels are good as the website consists of mostly black text on white background with the exception of the main news article on the front page. Here a low tone turquoise is used which maintains a good contrast ratio between text and background. No further investigation needed.

Resize text
Most of the text changes according to the text size. The site remains functional (images, functions, etc.) after the change of text size. Text is not cut off and does not overlap with other elements.

Keyboard access and visual focus

  • Tab to all: You are able to tab to all the different buttons, elements, etc.
  • Tab away: You are able to tab away from all buttons, elements.
  • Tab order: The tab order follows a logical order.
  • Visual focus: The selected element has a grey box around it.
  • All functionality by keyboard: You can access the different elements through a keyboard.
  • Dropdown-list: You cannot use the drop-down list when you are zoomed in only using a keyboard. It is mostly used for handheld mobile devices and is therefore not a major issue.
  • Image link: You can tab to the pictures, but no clear visual.
  • Forms, labels and errors: The available form (search) is accessible through keyboard. The <label> tag is not properly designed and lack a "for" and "id" tag. The labels are positioned correctly. The search field is not dependent on color, but also uses text to indicate its purpose. I could not find error messages for the search function.

Multimedia (video, audio) alternatives
No multimedia content available.

Basic structure check
The information makes sense in the order it is shown. The alternative text provides adequate information for the missing image The different sections have a clear heading, making it easier to read.

Comments
Under the "Timebestilling" page you are given the option to schedule an appointment via SMS or an online schema. When clicking the link you are redirected to an external website. This website seems like an outdated and lacking second website for the doctor's office. Having a second website with lacking and outdated content might cause confusion in the event of a patient discovering the second website first. A solution would be to implement the scheduling functionality on the primary website, or strip the second website of all other content.

The poor website - Hommelvik Legekontor

EasyChecks

Title
The titles are adequate and briefly describe the content of the page. You can distinguish the different pages based on their title.

Alt-text
Some images lack an alternative text.

Headings
<h2> and <h3> is used in the web pages according to the source code. The headings are used as proper headings. The web page seems to skip <h1> tags. Some of the text presented in the content can be viewed as headings, but do not carry the right code to validate it as such. All text that looks like a heading is not marked as a heading. The headings and the text is a mess.

Contrast
Contrast levels are mostly good as the website follows a template with black text on a low tone grey background. On the "Ansatte" page, however, the text background is of a darker grey which could cause problems for some users.

Resize text
Text resize only affects a few paragraphs such as the postal address on the front page and the bottom four links on the "Linker" page. Other than that the text size is unaffected. The page still remains functional (forms, buttons, pictures, etc.) and it does not overlap.

Keyboard access and visual focus

  • Tab to all: You are able to tab to all elements of the page.
  • Tab away: You are able to tab away from all the elements.
  • Tab order: The tab order follows a logic order.
  • Visual focus: Only some of the elements are clearly visible when selected.
  • All functionality by keyboard: You can access all elements through keyboard.
  • Drop-down lists: No drop-down list available.
  • Image links: No clear visual focus but you can access them through keyboard.
  • Forms, labels and errors: Couldn't find "label", "for" or "id" tags in the search form. No labels. No required field or other instructions. Could not create an error message through the search form.

Multimedia (video, audio) alternatives
No multimedia content available.

Basic structure check
The order of information makes somewhat sense. It's understandable, but far from perfect. The alternative text for pictures does not provide good enough description. Some of the blocks of text do not have a clear heading as it differs from one section to another.

Comments
The the user is given the option to schedule an appointment via a linked image in the sidebar on all of the websites pages. When clicking the image you are redirected to an external website where you are to log in with your personal information. The redirected website is clean and to the point. It would be an improvement to implement this functionality into their primary website, however, we suspect this to be impossible due to privacy policies in Norway.

Disposition for our website

Målet med siden, oppsummert i en eller to setninger
Siden skal være fungerende hjemmeside for et fiktivt legekontor. Den skal inneholde all informasjon som kan være nyttig for kontorets pasienter, og i tillegg støtte timebestillinger.

Antall sider med tittel for hver side
Planlagte sider:

  1. Hjem
  2. Nyheter
  3. Om oss
  4. Ansatte (Med «under sider» for hver ansatt)
  5. Kontoret tilbyr
  6. Priser
  7. Kontakt
  8. Timebestilling
    1. Bestill time på nett
  9. Nyttige lenker
  10. Sitemap

Potensielt publikum. Må du ta høyde for støtte til eldre nettlesere eller forskjellige oppløsninger?
De som skal lese siden vil være pasientene til legekontoret. Det vil si en målgruppe som strekker seg fra omtrent 12 år og oppover. Siden bør derfor støtte flest mulig nettlesere, også eldre, for at alle skal kunne ha tilgang. Siden bør også støtte mobile nettlesere, for at informasjonen skal være lett tilgjengelig.

Hvilket multimedieinnhold du skal ha med.
«Ansatte»-sidene vil inneholde bilder av legene, «Om oss»-siden vil ha et bilde av legekontoret, og eventuelt beskrivende bilder kan benyttes på nyhetssiden.

Hva slags interaksjon og dynamiske funksjoner skal du ha med?
Timebestilling på nett vil skje gjennom et skjema med tekst- og kryssbokser. Under «Kontakt oss» vil vi implementere Google Maps som brukeren selv kan operere. Navigasjonsbaren vil også inneholde en søkeboks som brukeren kan benytte til enhver tid.

En indikasjon fra hvor du får innholdet fra (inkludert tekst, bilder og andre filer).
Alt innhold vil være fiktivt og vil være en etterligning av en eller flere eksisterende sider for legekontor(er). Bildene vil være relevante, men vil være av andre fasiliteter eller områder.

Hva du lærte av å se på lignende sider
Det er mye bruk av dårlige maler. Store mangler i oversiktlighet og klarhet i informasjon er gjennomgående for mange legekontorers nettsider. Klarhet og tilgjengelighet er spesielt viktig med tanke på målgruppen, og vi ser her at vi må fokusere på dette.
